FLV封装格式简介

发表于 2016-10-20

FLV封装其特点在于支持实时观看，即无需下载整个文件，也可以通过接受视频中间的Tag数据来解析播放。
另外一个注意点: FLV中通过Tag区分不同的流，不像ts同一种类型支持多个流，FLV每种Tag最多存在一个流.
详细的文件格式介绍见下文:

##FLV文件头格式

  • 如图

  • 说明

    • 前三个字节是固定值，即文件标识“FLV”
    • Version: FLV格式的版本
    • TypeFlagsReserved: 固定为0
    • TypeFlagsAudio: 如果有音频Tags，则为1，否则为0
    • TypeFlagsReserved: 固定为0
    • TypeFlagsVideo: 如果有视频Tags，则为1，否则为0
    • DataOffset: FLV文件头长度。目前值固定为9，以后可能会扩充。

##FLV文件体

  • 如图

  • 说明

    • PreviousTagSize0: 固定为0，因为它前面无Tag数据
    • Tag1: 第一个Tag数据，Tag结构后面会详细说明
    • PreviousTagSize1: Tag1数据的大小，包括数据头。对于FLV Version1，因为数据头大小固定是11，所以该值为11+数据大小

##Tag定义
FLV Tag可以存储音频、视频、脚本数据、加密数据等。如图

  • 说明

    • Reserved: 预留给Flash Media Server，应该为0
    • Filter: 标明数据是否需要处理，0表示不需要，1表示需要（如加密）。
    • TagType: 标明Tag中的数据类型，其中8表示音频，9表示视频，18表示描述数据
    • DataSize: 数据大小，从StreamID到Tag结束，不包括StreamID
    • Timestamp: 时间戳，单位ms，与第一个Tag的时间戳相关，一般第一个Tag时间戳为0
    • TimestampExtended: 时间戳扩展，与Timestamp共同组成一个int32_t的值，扩展部分表示前8个bit
    • StreamID: 一般是0
    • AudioTagHeader: 如果Tagtype=8，则有此部分，详细结构见后面说明
    • VideoTagHeader: 如果Tagtype=9，则有此部分，详细结构见后面说明
    • EncyptionHeader: 如果Filter=1，则有此部分，此部分不做说明，详细信息可参考下面的参考文献
    • FilterParams: 如果Filter=1，则有此部分，此部分不做说明，详细信息可参考下面的参考文献
    • Data: 如果Tagtype=8，则此部分是音频数据，Tagtype=9，则此部分是视频数据，Tagtype=18，则为说明信息

  • 注意事项

    • Linear PCM表示数据存的是PCM裸数据，如果一个采样的大小为8bit，则数据直接存入unsigned bytes，如果一个采样大小为16bit，采样的储存采用Little endian。如果为双声道，则采样数据采用 left-right-left-right的形式储存。
    • SoundFormat 禁止使用0，建议使用3代替，原因为其在采样大小为16bit时，有设备的endian决定其endian
    • 如果SoundFormat 为AAC，那么SoundType必须为1（stereo），SoundRate必须为3（44kHz).而实际的音频采样率与声道信息由AAC数据提供

  • 音频数据结构介绍
    在音频Tag包中，主要有2部分需要介绍 —— AudioTagHeader与AudioData，如图

    • AudioTagHeader说明

      • SoundFormat:
        • 0 = Linear PCM, Platform endian
        • 1 = ADPCM
        • 2 = MP3
        • 3 = Linear PCM, little endian
        • 4 = Nellymoser 16kHz mono
        • 5 = Nellymoser 8kHz mono
        • 6 = Nellymoser
        • 7 = G.711 A-law logarithmic PCM
        • 8 = G.711 mu-law logarithmic PCM
        • 9 reserved
        • 10 = AAC
        • 11 = Speex
        • 14 = MP3 8kHz
        • 15 = Device-specific sound
      • SoundRate:
        • 0 = 5.5kHz
        • 1 = 11kHz
        • 2 = 22kHz
        • 3 = 44kHz
      • SoundSize: 单位音频采样的大小，0表示8bit, 1表示16bit
      • SoundType: 单声道或双声道，0表示单声道，1表示双声道
      • AACPacketType: SoundFormat==10时有，0表示AAC序列头，1表示AAC数据
    • AudioData说明：如果加密，则为加密的数据，如果未加密则为音频数据，具体编码格式由SoundFormat确定

  • 视频数据结构介绍
    与音频数据类似，视频Tag也主要有两部分需要介绍 – VideoTagHeader与VideoData，如图

    • VideoTagHeader说明

      • FrameType: 表示视频帧类型，有如下定义
        • 1 = 关键帧（for AVC， 不需要参考其它帧，跳到此帧可以直接播放）
        • 2 = 非关键帧 （for AVC，需要参考其它帧，跳到此帧不能直接播放）
        • 3 = 一次性非参考帧 （h263 only）
        • 4 = generated key frame (预留给服务用的)
        • 5 = 视频信息/命令帧
      • CodecID:
        • 2 = Sorenson H.263
        • 3 = Screen video
        • 4 = On2 VP6
        • 5 = On2 VP6 with alpha channel
        • 6 = Screen video version 2
        • 7 = AVC
      • AVCPacketType: CodecID为7时有此数据
        • 0 = AVC 序列头，即解码信息（AVCDecoderConfigurationRecord）
        • 1 = AVC 网络数据单元（NALU），可以有多个NALUs
        • 2 = AVC 尾部序列
      • CompositionTime: CodecID为7时有此数据，AVCPacketType=1时表示时间偏移，否则为0
    • VideoData说明：如果加密，则为加密的数据，如果未加密则为音频数据，具体编码格式由CodecID确定

  • 脚本数据
    脚本数据如果被加密，则为加密数据，如果未加密则按照如下规则组织数据：

      • 每一个脚本数据的数据体都由两部分组成 – Name和Type，它们的数据组织都是SCRIPTDATAVALUE
        • Name: 方法或者对象的名字
        • Type: Adobe Media Flush参数或者对象属性
      • SCRIPTDATAVALUE的结构：
        • Type: SCRIPTDATAVALUE数据类型
          • 0 = Number
          • 1 = Boolean
          • 2 = String
          • 3 = Object
          • 4 = MovieClip (reserved, not supported)
          • 5 = Null
          • 6 = Undefined
          • 7 = Refence
          • 8 = ECMA array
          • 9 = Object end marker
          • 10 = Strict array
          • 11 = Date
          • 12 = Long string
        • ScriptDataValue: 由Type决定，对应关系如下：（详细数据组织形式参考标准文档）
          • 0 – DOUBLE
          • 1 – 8bit
          • 2 – SCRIPTDATASTRING
          • 3 – SCRIPTDATAOBJECT
          • 7 – 16bit
          • 8 – SCRIPTDATAECMAARRAY
          • 10 – SCRIPTDATASTRICTARRAY
          • 11 – SCRIPTDATADATE
          • 12 – SCRIPTDATALONGSTRING
